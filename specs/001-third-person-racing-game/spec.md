## User Scenarios & Testing *(mandatory)*

### User Story 1 - 單人計時賽 (Priority: P1)

玩家選擇賽車與賽道後，以第三人稱視角進行計時賽，目標是在最短時間內完成全部圈數。遊戲全程以第三人稱鏡頭跟隨賽車，玩家可透過鍵盤或手把控制加速、煞車與轉向。

**Why this priority**: 計時賽是賽車遊戲最核心的可玩性，提供即時可驗證的 MVP 體驗，所有其他模式均建立在此基礎之上。

**Independent Test**: 可單獨啟動遊戲 → 選擇賽車 → 選擇賽道 → 完成一圈計時賽，驗證計時器正確運作並顯示完賽時間。

**Acceptance Scenarios**:

1. **Given** 玩家在主選單，**When** 選擇「開始遊戲」→ 選擇賽車 → 選擇賽道，**Then** 遊戲進入賽道場景，顯示第三人稱視角的賽車，計時器從 0 開始倒數準備。
2. **Given** 賽局進行中，**When** 玩家按下加速鍵，**Then** 賽車向前加速，速度計顯示數值上升，第三人稱鏡頭隨車移動。
3. **Given** 賽局進行中，**When** 玩家完成所有圈數過終點線，**Then** 計時停止，畫面顯示完賽時間與最快單圈時間，並提示玩家可重試或返回選單。
4. **Given** 玩家完賽，**When** 該次成績優於歷史紀錄，**Then** 系統更新並顯示新的個人最佳紀錄。

---

### User Story 2 - 賽車與賽道選擇 (Priority: P2)

玩家在開始遊戲前，可從多輛賽車與多條賽道中進行選擇，並看到各選項的基本屬性（如速度、操控性、賽道難度）。

**Why this priority**: 選擇賽車與賽道是進入核心遊戲循環的必經步驟，提升玩家的參與感與重玩價值。

**Independent Test**: 可單獨測試選擇介面，驗證所有賽車與賽道的縮圖、名稱及屬性均正確顯示，且選擇後能正確進入對應場景。

**Acceptance Scenarios**:

1. **Given** 玩家在選擇畫面，**When** 瀏覽賽車清單，**Then** 每輛賽車顯示名稱、縮圖與速度／操控性評分。
2. **Given** 玩家選定賽車，**When** 瀏覽賽道清單，**Then** 每條賽道顯示名稱、縮圖與難度標示。
3. **Given** 玩家選好賽車與賽道，**When** 按下「確認」，**Then** 遊戲載入並進入對應場景。

---

### User Story 3 - 碰撞與賽道邊界處理 (Priority: P3)

當賽車撞到障礙物或衝出賽道邊界時，遊戲給予明確的視覺／音效回饋，並讓賽車以合理方式恢復位置，避免玩家卡關。

**Why this priority**: 碰撞回饋直接影響遊戲手感與公平性，但可在核心計時賽 MVP 之後疊加完善。

**Independent Test**: 可單獨測試賽車故意衝出邊界的情境，驗證賽車被重置回賽道且計時繼續，玩家不需重啟遊戲。

**Acceptance Scenarios**:

1. **Given** 賽局進行中，**When** 賽車撞上賽道邊界或障礙物，**Then** 播放碰撞音效，賽車速度降低，並顯示輕微震動效果。
2. **Given** 賽車完全衝出賽道範圍，**When** 超過邊界容忍距離，**Then** 系統在 3 秒內將賽車重置回最近的賽道起點，計時繼續累計。
3. **Given** 賽車被重置，**When** 玩家恢復操控，**Then** 賽車方向朝向賽道正確前進方向。

---

### User Story 4 - 主選單與遊戲暫停 (Priority: P4)

玩家可以從主選單啟動遊戲、查看個人最佳紀錄，以及在賽局中隨時暫停並選擇繼續、重試或返回主選單。

**Why this priority**: 選單與暫停是完整遊戲體驗的基本框架，但屬於包裝功能，核心玩法優先。

**Independent Test**: 可單獨驗證主選單所有按鈕均可操作，暫停選單在賽局中正確凍結遊戲並提供選項。

**Acceptance Scenarios**:

1. **Given** 玩家開啟遊戲，**When** 進入主選單，**Then** 顯示「開始遊戲」、「查看紀錄」與「離開」選項。
2. **Given** 賽局進行中，**When** 玩家按下暫停鍵，**Then** 遊戲凍結，顯示「繼續」、「重試」、「返回主選單」三個選項。
3. **Given** 暫停選單開啟，**When** 選擇「重試」，**Then** 重新載入同一賽道與賽車，計時器歸零重新開始。

---

### Edge Cases

- 玩家在賽局開始前 3 秒倒數結束前強行移動時，賽車不應提前移動。
- 當圈數設定為 1 圈且玩家剛好在終點線起步時，系統應正確辨識完賽條件（需完整行駛一圈）。
- 玩家在網頁版中切換分頁或最小化視窗時，遊戲應自動暫停。
- 成績紀錄儲存失敗（如瀏覽器隱私模式禁止儲存）時，應顯示提示訊息而非靜默失敗。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統 MUST 提供第三人稱跟隨鏡頭，鏡頭始終跟隨玩家賽車並保持固定後方視角。
- **FR-002**: 玩家 MUST 能透過鍵盤方向鍵（或 WASD）控制賽車的加速、煞車與左右轉向。
- **FR-003**: 系統 MUST 顯示即時速度計與圈數計時器於遊戲畫面。
- **FR-004**: 系統 MUST 在玩家完成全部圈數後停止計時並顯示完賽成績。
- **FR-005**: 系統 MUST 將玩家個人最佳成績儲存於本地端，並在選單與完賽畫面中顯示。
- **FR-006**: 系統 MUST 提供至少 3 輛可選賽車，每輛具有不同的速度與操控性屬性。
- **FR-007**: 系統 MUST 提供至少 2 條可選賽道，每條賽道具有不同的佈局與難度。
- **FR-008**: 系統 MUST 在賽車碰撞邊界或衝出賽道時提供視覺與音效回饋。
- **FR-009**: 系統 MUST 在賽車完全離開賽道後自動將其重置回最近的賽道起點。
- **FR-010**: 玩家 MUST 能在賽局進行中隨時暫停，並選擇繼續、重試或返回主選單。
- **FR-011**: 系統 MUST 在賽局開始前顯示 3 秒倒數計時，倒數期間賽車不可移動。
- **FR-012**: 遊戲 MUST 可在瀏覽器中執行並透過靜態網頁部署（可部署至 GitHub Pages）。

### Key Entities

- **賽車（Car）**: 代表玩家操控的車輛，具有名稱、速度上限、操控性、外觀縮圖等屬性。
- **賽道（Track）**: 代表比賽場景，具有名稱、難度、圈數設定、起跑位置、賽道邊界定義。
- **成績紀錄（Record）**: 代表玩家在特定賽道的最佳完賽時間，關聯至賽道與賽車。
- **賽局（Race Session）**: 代表一次完整的比賽過程，記錄當前圈數、計時、賽車狀態。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 玩家從主選單出發，完成選擇賽車與賽道並進入賽局，全流程在 30 秒內完成。
- **SC-002**: 賽車控制的輸入延遲不超過 100 毫秒，確保操控即時響應。
- **SC-003**: 遊戲畫面維持穩定，主流桌面瀏覽器中不出現明顯卡頓或閃爍。
- **SC-004**: 90% 的測試玩家在不閱讀說明的情況下，能在首次遊玩的 2 分鐘內完成一場計時賽。
- **SC-005**: 玩家完賽後，個人最佳紀錄正確更新並在下次遊玩時仍可顯示（本地端持久化）。
- **SC-006**: 賽車碰撞或衝出賽道後，3 秒內自動恢復至可繼續遊玩的狀態，不需玩家手動重置。

## Assumptions

- 遊戲為單人單機體驗，不包含多人連線對戰功能。
- 遊戲以網頁形式運行，支援主流桌面瀏覽器（Chrome、Firefox、Edge、Safari）。
- 賽道為封閉式環形賽道，具有明確的起終點線。
- 預設圈數為 3 圈，玩家在選擇賽道時可見圈數資訊但不可自訂。
- 遊戲不包含賽車升級或付費系統，所有賽車與賽道從一開始均可選用。
- 音效為選配功能，若瀏覽器自動播放政策阻擋時，遊戲仍可正常運作。
- 個人最佳紀錄以瀏覽器本地儲存方式儲存，清除瀏覽器資料後紀錄將消失，此為預期行為。
